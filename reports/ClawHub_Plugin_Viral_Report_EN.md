## Executive Summary

The ClawHub plugin market is still early. On April 20, 2026, the public plugin detail payloads expose almost no meaningful 'downloads', 'installs', or 'stars', so plugin breakout analysis has to rely on ranking position, verification quality, plugin structure, scan coherence, and author production systems rather than consumer popularity numbers alone.

## What Actually Wins

1. Code plugins outrank bundle-only packaging.
2. Strong plugin names describe the system or operational job directly.
3. Source-linked verification is a major trust accelerator.
4. The best authors ship adjacent plugins in one clear lane instead of random one-offs.
5. Security, governance, growth ops, payments, storage, and workflow connectors are the best AISA-conversion lanes.

## Repeatable Playbook

- Ship plugins that connect to a concrete external system or decision surface.
- Explain side effects and setup paths clearly.
- Keep plugin scope tight enough that verification reads as coherent.
- Turn one high-value lane into a multi-plugin factory.
- Package for trust, not just for parser correctness.
