## 一句话卖点

给我一个 GitHub 仓库链接，我在一轮内告诉你这个项目是做什么的、值不值得继续看、核心文件在哪、依赖什么、有没有明显风险。

## 简介长文案

GitHub Repo Research 是一个面向开发者、独立黑客、技术运营和投研场景的仓库分析 skill。它不做泛泛而谈的 README 复述，而是把一个 GitHub 链接快速转成结构化决策简报：项目用途、目标用户、技术栈、关键入口文件、外部 API 或服务依赖、成熟度信号、复用价值和下一步建议。

适合你在以下场景使用：

- 看一个陌生仓库前，先判断值不值得投入更多时间。
- 找竞品或同类实现时，快速筛掉不值得深挖的 repo。
- 做技术选型、开源接入或方案参考时，先拿到一份短平快的结论。
- 让 agent 在进入代码级分析前，先完成仓库级尽调。

## 三条展示文案

- 用一个 GitHub 链接，换一份能直接判断“看不看、抄不抄、接不接”的仓库简报。
- 不是代码审计，也不是 README 复述，而是面向行动的 repo 研究结果。
- 非常适合做开源项目筛选、竞品扫描、技术选型和 agent 前置判断。
