# Launch Plan

## Day 1

- 锁定输入格式：只收 GitHub repo URL。
- 锁定第一版输出结构：结论、用途、技术栈、入口文件、依赖、风险、下一步。
- 准备 3 个公开示例 repo 作为演示样本。

## Day 2

- 打磨标题和描述，用 GitHub / Repo / Research / Analyzer / Summary 这类搜索词。
- 把第一轮回答控制成可以 30 秒扫完的结构。
- 明确“推断”和“已验证”两种表述方式。

## Day 3

- 验证 3 个不同类型仓库：
  - SDK / 框架
  - AI 工具
  - 自动化或 CLI 项目
- 修正输出里最容易空泛的部分。

## Day 4

- 补 3 个 example prompts。
- 补“何时使用 / 何时不要使用”。
- 补最小商业化钩子。

## Day 5

- 发布第一版。
- 收集用户最常问的追加问题。
- 记录哪些字段最常被引用。

## Day 6-7

- 从核心 skill 扩出 2 个变体：
  - GitHub PR Review
  - GitHub Tech Stack Audit
- 复用同一套分析结构，不换底层能力。
