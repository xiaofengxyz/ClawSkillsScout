# ClawHub 爆款技能工厂手册

## 核心原则

- 先定义任务，再定义功能。
- 先追求首轮成功，再追求功能完备。
- 先做单点场景，再做作品集扩张。
- 先验证下载与复用，再加复杂自动化。

## 爆款结构

- 标题：直接对应搜索词
- 描述：做什么、什么时候用、支持什么
- 输入：尽量少
- 输出：尽量确定
- 示例：第一轮就能跑通
- 定价钩子：实时性、配额、批量、自动化深度

## 从 0 到 1

1. 选一个高需求 API 家族
2. 写一句话任务定义
3. 设计第一个成功路径
4. 补上标题、描述、示例
5. 设计 5 个相邻变体
6. 发布并记录下载/复用/纠错
7. 把赢家抽成模板

## 普通 skill 变爆款

- 砍掉大而全描述
- 把名字改成用户会搜的词
- 缩短输入链路
- 提高输出确定性
- 强化“什么时候用”
- 增加一个清晰的升级理由

## 高产作者方法论

- 他们不是在重复造新产品，而是在复用同一个底层能力。
- 他们不是一次性赌大爆款，而是在做多变体组合。
- 他们把标题、结构、输出格式做成模板。
- 他们把发布当成持续试验，而不是单次上线。
