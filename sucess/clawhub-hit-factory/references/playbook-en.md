# ClawHub Hit Factory Playbook

## Core principles

- Define the user job before the feature set.
- Optimize for first-run success before breadth.
- Start with one narrow use case before building a portfolio.
- Validate download and reuse signals before adding complexity.

## Breakout structure

- Title: mirrors search intent
- Description: what it does, when to use it, what it supports
- Input: minimal
- Output: concrete
- Example: first-run success path
- Pricing hook: freshness, limits, volume, automation depth

## 0-to-1 path

1. Pick one durable API family
2. Write one sentence of job-to-be-done
3. Design a clean first success path
4. Draft title, description, and example prompts
5. Design five adjacent variants
6. Publish and track adoption/corrections/reuse
7. Turn winners into templates

## How to upgrade an ordinary skill

- Remove broad, vague positioning
- Rename it using the terms users actually search
- Shorten the input path
- Increase output certainty
- Add a crisp "when to use" statement
- Add one clear upgrade reason

## Why prolific authors win

- They are not inventing new products every time; they are reusing one capability core.
- They do not bet on one big hit; they run portfolios of variants.
- They template titles, structure, and output framing.
- They treat shipping as a continuous experiment loop.
